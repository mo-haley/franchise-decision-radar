# Phase 3 Site Plan — Franchise Decision Radar v1

**Date:** 2026-03-14

---

## Architecture

Static HTML site generated by a Python build script using Jinja2 templates.
No JavaScript frameworks. Minimal vanilla JS for interactive elements (revenue
level toggle on fee page). All CSS inline in the base template.

### Data flow

```
data/extracted/*.json ──┐
                        ├──▶ scripts/build_site.py ──▶ site/*.html
data/derived/*.json ────┘         │
                            templates/*.html
```

The build script reads structured JSON artifacts, assembles a unified data context,
and renders Jinja2 templates to static HTML in `site/`.

### File structure

```
templates/
  base.html              ← shared layout, nav, CSS, notation legend
  index.html             ← homepage
  fee-burden.html        ← fee burden comparison page
  system-health.html     ← system health trajectory page
  cost-to-enter.html     ← cost to enter page

scripts/
  build_site.py          ← reads data, renders templates to site/

site/
  index.html             ← generated
  fee-burden.html        ← generated
  system-health.html     ← generated
  cost-to-enter.html     ← generated
```

## Pages

### 1. Homepage (`index.html`)
- What Franchise Decision Radar is
- The mosquito franchise cohort scope
- Trust posture: regulator-source, explicit modeling, caveat-aware
- Links to the 3 comparison pages
- Notation system legend

### 2. Fee Burden Comparison (`fee-burden.html`)
- Revenue-level selector ($200K / $300K / $400K / $500K)
- Comparison table: brand × total burden, % of revenue
- Component breakdown: royalty, marketing, tech+other
- Key insight callout: marketing > royalty as cost driver
- Inline caveats per brand where model has uncertainty
- Rank-order shift note at $500K

### 3. System Health Trajectory (`system-health.html`)
- System size comparison (end 2024)
- Net unit growth table (2022–2024 + 3-year total)
- Attrition table (terminations + non-renewals + ceased, as count and %)
- Growth trajectory assessment per brand
- Red flag callouts (Shield churn, Joe termination spike)
- Caveat on closure classification differences

### 4. Cost to Enter (`cost-to-enter.html`)
- Franchise fee comparison
- Total initial investment range comparison
- Investment midpoint comparison
- Squad 12-month vs 3-month caveat (prominent)
- Insight: low upfront ≠ low long-term burden (link to fee page)
- Royalty structure overview

## Notation System

Consistent labels used across all pages:

| Label | Meaning | Visual |
|---|---|---|
| Extracted | Value directly from the FDD | Gray tag |
| Modeled | Computed from FDD terms using explicit assumptions | Blue tag |
| Editorial | Assessment based on documented evidence | Amber tag |
| Caveat | Limited comparability or model uncertainty | Bordered callout |

---

## Design Principles

- Clean, restrained color palette (near-black text, white background, muted accents)
- Strong tables as primary data presentation
- Callout blocks for insights and caveats (not hidden in footnotes)
- No decorative elements, no stock imagery
- Monospace for numbers, sans-serif for text
- Mobile-readable but desktop-primary

---

## Intentionally Excluded from v1

- Individual brand detail pages
- Item 19 revenue comparisons (not yet comparable)
- Profitability estimates
- Rankings or scores
- Interactive charts or visualizations
- Search, filtering, or sorting
- Any form of "recommendation" or "best franchise" language
- SEO optimization, analytics, deployment infrastructure
